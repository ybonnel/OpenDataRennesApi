Données téléchargeables

Données théoriques du réseau STAR  

Les données sont disponibles au format GTFS. La documentation de format ainsi que des des ressources pour les développeurs sont disponibles sur le site officiel décrivant le format (Lien : http://code.google.com/intl/fr/transit/spec/transit_feed_specification.html)

Liste des fichiers présent dans le GTFS :

Depuis la mise à jour du 11 juillet 2011, l’ensemble des valeurs dans les fichiers CSV sont entourés par des guillemets, afin de prévenir l’utilisation future du champ de séparation CSV.


agency.txt
Aucune remarque particulière.

calendar.txt
Aucune remarque particulière.

calendar_dates.txt
Aucune remarque particulière.

feed_info.txt
Depuis la mise à jour du 14/11/2011, ce fichier (présent depuis la mise à jour de la spécification GTFS du 06/09/2011) est inclus dans le lot GTFS.


routes.txt :
Le champ route_desc contient la description de la catégorie de la ligne (Majeure, Express, ...)

routes_extensions.txt :
Ce fichier d'extension à la norme GTFS contient des informations supplémentaires sur les routes définies dans le fichier routes.txt.
Le champ route_accessible indique si la ligne est considérée comme une ligne accessible aux Personnes à Mobilité Réduite (PMR) (0 = NON ; 1 = OUI). Attention : pour qu'une PMR puisse prendre la ligne indiquée comme accessible, il faut que l'arrêt soit également accessible (voir le fichier stop_extensions.txt).

stop_times.txt :
Aucune remarque particulière

stops.txt :
Le champ stop_desc contient la commune de rattachement de l'arrêt (au sens de l'INSEE).

stops_extensions.txt :
Ce fichier d'extension à la norme GTFS contient des informations supplémentaires sur les arrêts définis dans le fichier stops.txt.
Le champ stop_accessible indique que l'arrêt est accessible aux Personnes à Mobilité Réduite (PMR) (0 = NON ; 1 = OUI). Attention : pour qu'une PMR puisse prendre une ligne à un arrêt accessible, il faut que la ligne soit également accessible (voir le fichier routes_extensions.txt).
Le champ stop_bench indique que l'arrêt dispose d'un banc (0 = NON ; 1 = OUI). Le champ stop_lighted indique que l'arrêt est éclairé la nuit (0 = NON ; 1 = OUI). Enfin, le champ stop_sheltered indique que l'arrêt est couvert (0 = NON ; 1 = OUI).

trips.txt :
Aucune remarque particulière
